## Summary

Owner D delivers the client-facing and devops vertical slice of Concierge: a signed-token-authenticated embeddable React widget, a Streamlit admin config page for tenant operators, and a GitHub Actions CI/CD pipeline with six gated eval suites. The hard constraint in every deliverable is isolation — the widget token is the sole source of `tenant_id`, and every CI gate must block merge on regression.

---

## Technical Context

**Language/Version**: Python 3.11 (FastAPI backend + Streamlit admin + eval scripts), TypeScript/React 18 (widget)

**Primary Dependencies**: FastAPI, PyJWT, Streamlit, Vite, React, GitHub Actions

**Storage**: PostgreSQL (widgets, tenant_config, widget_sessions tables), Redis (CORS cache TTL 60s, widget sessions TTL 1800s)

**Testing**: pytest (Python backend), Vitest (React widget), GitHub Actions CI

**Target Platform**: Docker (Linux containers), Browser (widget)

**Project Type**: Multi-tenant web service + embeddable frontend widget + admin dashboard

**Performance Goals**: Widget bundle < 50 KB gzipped; token exchange < 100ms p95; widget load-to-ready < 1s

**Constraints**: No torch in any container; widget iframe must not break host page on load failure; tenant_id must never come from request body

**Scale/Scope**: 2 demo tenants for submission; architecture handles 1,000 tenants (isolation by RLS + token)
